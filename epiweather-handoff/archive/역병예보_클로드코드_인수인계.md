## 프로젝트 한 줄 요약

**역병예보 (EpiWeather)** — 합성위협 탐지 + 전세계 28개 언어 신호망 + AI 갭필링을 결합한 감염병 조기경보 플랫폼. 한국 질병청 실데이터와 네이버 검색 트렌드가 이미 연결되어 있고, Windows 작업 스케줄러로 시계열 데이터를 쌓는 중.

## 알려진 한계 (정직하게 공유)

1. **WHO EMRO·WPRO RSS 자체가 운영 중단됨** — Claude/Perplexity 검색으로 우회 중
2. **ReliefWeb API는 사전승인 appname 필요** (2025.11~) — `reliefweb.int/api-users` 신청 필요
3. **MSF.org는 CloudFlare 보호로 직접 스크래핑 불가** — AI 검색으로 우회 중
4. **AI가 가끔 날짜를 혼동함** (예: 예멘 콜레라 검색 시 2017년 자료를 2026년인 것처럼 반환한 사례 있음) → 응답에 연도 명시 강제 + 교차검증 로직 필요
5. **KDCA 데이터가 아직 연간 집계 수준** — 교차상관 분석이 유의미해지려면 주간 데이터 누적이 더 필요함 (스케줄러가 이걸 메우는 중)

## Claude Code 통합 과제

### Phase 1 — 백엔드 (최우선)
```
1. pipeline_v2.py + epiweather_master_connector.py + 역병예보_AI감시.py
   → FastAPI 단일 앱으로 통합
2. 자동감시_허브.py의 로직을 APScheduler 또는 Celery beat로 이식
   (Windows 작업 스케줄러는 임시 다리, 서버에서는 진짜 스케줄러 사용)
3. signals_log.jsonl 데이터를 SQLite/PostgreSQL로 마이그레이션
4. REST API:
   GET /api/status      - 전체 시스템 상태
   GET /api/signals      - 최신 글로벌 신호 (free + AI)
   GET /api/timeseries   - 시계열 데이터 (스케줄러가 쌓은 것)
   GET /api/threats      - 현재 위협 목록 (에볼라 PHEIC 등)
   GET /api/kdca         - 국내 ILI 데이터
   GET /api/naver        - 검색 트렌드
   POST /api/alerts      - 경보 발령 (카카오/이메일 webhook)
```

### Phase 2 — 프론트엔드
```
1. epiweather-stage0-v3.html 등 기존 HTML들의 디자인 언어를 React로 재구성
2. 실시간 대시보드: WebSocket으로 signals_log 업데이트 푸시
3. 모바일 반응형
```

### Phase 3 — 배포
```
백엔드: Railway.app 또는 Render.com (~$5/월)
프론트: Vercel (무료)
DB: Supabase (무료~$25/월)
도메인: epiweather.kr
```
