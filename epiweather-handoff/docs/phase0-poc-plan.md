# Phase 0 PoC 실행계획
### 추론 코어 → 실제 데이터 연결 · 90일 작전

> 목표는 단 하나의 증거다: **"실제 데이터로, 하나의 질병을, 공식 발표보다 N일 먼저 잡았다"** 를 재현 가능하게 입증한다. 이 증거가 투자·파트너·정부과제·신뢰의 모든 문을 여는 열쇠다. 거창한 플랫폼이 아니라, 작고 단단한 첫 승리를 만든다.

---

## 0. 한 줄 정의

후향(retrospective) 데이터로 **인플루엔자**를 대상으로, 우리 추론 엔진(Farrington 이상탐지 + 다신호 융합)이 질병관리청 공식 표본감시 발표보다 며칠 먼저 유행 시작을 탐지했음을 증명한다.

왜 후향인가: 과거 데이터는 "정답"(실제 유행이 언제 시작됐는지)이 이미 알려져 있어, 우리 예측이 맞았는지 즉시 채점할 수 있다. 실시간보다 빠르고 싸고 위험이 없다.

---

## 1. 왜 인플루엔자 × 후향인가 (대상 선정 근거)

- **데이터가 가장 풍부하다**: 인플루엔자는 질병관리청 표본감시(ILI, 인플루엔자 의사환자분율)가 주간 단위로 수년치 공개되어 있다. 베이스라인·계절성·정답이 모두 확보된다.
- **계절성이 뚜렷하다**: 매년 겨울 유행이 반복돼, 베이스라인 학습과 이상탐지 검증이 깔끔하다.
- **선행 신호가 존재한다**: 독감 관련 검색어(네이버·구글 트렌드)가 공식 ILI보다 앞선다는 건 국내외에서 반복 확인됐다(구글 독감 트렌드의 교훈 포함).
- **위험·규제 부담이 낮다**: 집계·공개 데이터 중심이라 민감정보 이슈가 적다.

> 신종(Disease X) 탐지가 궁극 목표지만, 신종은 정의상 과거 정답이 없어 PoC 검증이 불가능하다. **먼저 인플루엔자로 "선행 탐지 능력"을 입증한 뒤, 그 능력을 신종으로 확장**하는 순서가 옳다.

---

## 2. 데이터 소스 (Phase 0 최소 조합)

가장 빠르고 싸게 시작할 수 있는 3종으로 한정한다. 욕심내지 않는다.

| 순번 | 데이터 | 역할 | 확보 방법 | 비용/난이도 |
|---|---|---|---|---|
| ① | **질병관리청 ILI 표본감시** | 정답·베이스라인 | 감염병포털·공공데이터포털 Open API | 무료 / 낮음 |
| ② | **검색어 트렌드** (독감·기침·해열제 등) | 선행 신호 A | 네이버 데이터랩 API, 구글 트렌드 | 무료 / 낮음 |
| ③ | **OTC/약국 판매 또는 응급실 ILI** | 선행 신호 B (택1) | 공개 통계 우선, 추후 제휴 | 무료~중 / 중 |

- ①②만으로도 PoC 1차 검증이 가능하다. ③은 가능하면 추가해 융합 효과를 보강한다.
- **하수 역학**은 선행성이 가장 좋지만 데이터 확보에 제휴·시간이 필요하므로 Phase 1로 미룬다. (단, 가능성 타진은 Phase 0에서 시작)

---

## 3. 90일 타임라인

### 🗓 Week 1–2 · 데이터 수집 셋업
- 공공데이터포털·감염병포털에서 ILI 주간 데이터 수년치 확보(②검색어와 기간 정렬)
- 네이버 데이터랩 / 구글 트렌드 API 키 발급, 독감 관련 검색어 후보군 30~50개 수집
- 데이터 파이프라인 구축: 수집 → 정제 → 주간 정렬 → 저장(CSV/DB)
- **산출물**: 정렬된 다년치 시계열 데이터셋 1벌

### 🗓 Week 3–4 · 신호 선별·정렬
- 검색어 후보 중 ILI와 상관·선행성이 높은 키워드 선별(교차상관 분석)
- 과적합 방지: 학습/검증 기간 분리(예: 과거 3시즌 학습, 직전 1~2시즌 검증)
- 구글 독감 트렌드의 실패 교훈 반영: 검색어만 맹신하지 말고 공식 데이터와 앵커링
- **산출물**: 선행 지표 후보 확정 + 상관/선행 리드 분석표

### 🗓 Week 5–8 · 엔진 연결·튜닝 (핵심 구간)
- 현재 프로토타입의 추론 코어(Farrington 이상탐지 + 융합 + Rt 추정)에 **실제 데이터 주입**
- 합성 데이터 → 실데이터로 교체. 베이스라인 학습·임계 튜닝
- 각 과거 시즌에 대해 "엔진이 며칠에 유행 시작을 탐지했는가" vs "공식 발표일" 비교
- **산출물**: 시즌별 리드타임 표 (엔진 탐지일 − 공식 발표일)

### 🗓 Week 9–11 · 검증·정량화
- 핵심 지표 산출:
  - **평균 선행 리드타임**(일) — "공식보다 평균 며칠 빨랐나"
  - **민감도/특이도** — 유행을 놓치지 않았나(미탐지), 헛경보는 없었나(오경보)
  - **방법 비교** — Farrington vs 단순 z vs EWMA 성능 (프로토타입에서 이미 토글 구현)
- 통계적 robustness: 여러 시즌·여러 random seed로 재현성 확인
- **산출물**: 검증 리포트 (정답이 있는 데이터에서의 성능 증거)

### 🗓 Week 12–13 · 패키징
- 한 장 결과 요약 + 재현 가능한 코드/노트북
- 데모: 프로토타입에 실데이터 결과를 얹어 "공식보다 N일 빨랐다"를 시각적으로 제시
- 다음 단계 제안서(Phase 1: 하수 역학 추가, 실시간 전환, 첫 B2B 파일럿)
- **산출물**: 투자/과제/파트너용 PoC 증거 패키지

## 4. 성공 기준 (정직한 합격선)

PoC는 다음을 충족하면 성공으로 본다:

- ✅ **선행성**: 최소 1개 이상의 과거 시즌에서, 엔진이 공식 ILI 경보 발표보다 **유의하게(예: ≥1주) 먼저** 유행 시작을 탐지
- ✅ **재현성**: 여러 시즌·여러 설정에서 일관된 선행 탐지(우연이 아님)
- ✅ **오경보 통제**: 비유행기 헛경보율이 실용 가능한 수준(예: 시즌당 1회 이하)
- ✅ **방법 우위**: Farrington 융합이 단순 단일신호보다 낫다는 정량 증거

> 실패해도 가치가 있다: "어떤 신호가 효과 없었는지"를 알게 되면 Phase 1 설계가 정교해진다. PoC의 목적은 성공 보장이 아니라 **빠르고 싸게 진실을 확인**하는 것이다.

---

## 5. 필요 자원 (최소)

- **사람**: 데이터/ML 엔지니어 1 + 역학·공중보건 자문 1(파트타임 가능). 역학 자문이 "유행 시작 정의·검증 기준"의 타당성을 보증한다 — 이게 결과의 신뢰성을 좌우한다.
- **인프라**: 노트북 + 클라우드 약간. 후향 PoC는 대규모 GPU 불필요.
- **비용**: 대부분 무료 공개데이터. 핵심 비용은 인건비·시간.
- **기간**: 약 90일(3개월).

이 단계는 **자본보다 시간·전문성 집약적**이다. 신약개발과 달리 웻랩·임상이 없어 초기 진입이 가볍다는 이 분야의 강점이 그대로 발휘된다.

---

## 6. 리스크와 대응

- **검색어 신호의 불안정성**(구글 독감 트렌드 교훈): 검색어만 의존하지 않고 공식 데이터에 앵커링, 다신호 융합으로 보완.
- **데이터 정렬·결측**: 주간/일간 단위 불일치, 공휴일 효과 → 정제 파이프라인에서 표준화.
- **과적합**: 학습/검증 기간 엄격 분리, 미래 데이터 누수 금지.
- **"후향이라 쉬웠던 것 아니냐"는 비판**: 검증 기간을 학습에서 완전히 격리해, 실시간과 동일한 조건을 모사.

---

## 7. Phase 0 이후 (이 증거로 무엇을 여는가)

PoC 증거 패키지가 나오면 곧바로:
- **Phase 1 진입**: 하수 역학·동물 예찰 데이터 추가, 실시간 파이프라인 전환, 시·군·구 단위 확장
- **첫 B2B 파일럿**: 병원 1곳 또는 지자체와 "응급실 수요 예측" 파일럿 (프로토타입의 B2B 패널이 그 가치 제안)
- **정부 R&D 과제 지원**: 질병청이 공표한 감염병 예측 고도화 수요와 직접 연결
- **시드 투자**: 이제 투자자가 요구하는 "검증된 예측 실적"을 손에 쥐고 협상

---

## 8. 이번 주 당장 할 일 (실행 트리거)

1. 공공데이터포털·감염병포털에서 ILI 표본감시 데이터 내려받기 (반나절)
2. 네이버 데이터랩 API 키 발급, 독감 검색어 10개로 1년치 추세 뽑아보기 (반나절)
3. 둘을 같은 그래프에 겹쳐 그려, **검색어가 ILI보다 앞서는지 눈으로 확인** (하루)

이 3가지를 끝내면 PoC의 "신호가 존재하는가"라는 가장 근본적 가설을 며칠 만에 검증할 수 있다. 여기서 선행성이 보이면, 나머지 89일은 그것을 단단한 증거로 만드는 과정이다.
